# UX Review - DrawNGuess (Telestrations)
**Date:** February 2, 2026  
**Reviewer:** UX Consultant  
**Platform:** Mobile-First Web App  
**Framework:** React + Socket.io

---

## Executive Summary

DrawNGuess is a multiplayer drawing and guessing game (digital Telestrations) with a **solid mobile-first foundation**. The app demonstrates strong technical implementation with real-time synchronization, responsive layouts, and modern UI patterns. However, there are **critical UX gaps** that could significantly impact player experience, particularly around **onboarding, feedback clarity, and mobile interaction patterns**.

**Priority Rating:** This review focuses on **P0 (Critical)** and **P1 (High)** issues that should be addressed before broad deployment.

---

## üéØ Findings by Screen

### 1. Landing Page

#### ‚úÖ Strengths
- Clean, uncluttered design with clear primary actions (Host/Join)
- Auto-generated player names reduce friction
- "How to Play" button provides access to rules
- Good use of visual hierarchy with gradient text
- Responsive grid layout adapts well to mobile

#### ‚ö†Ô∏è Critical Issues (P0)

**1.1 - Missing Game Description**
- **Issue:** No context about what "DrawNGuess" is for first-time users
- **Impact:** Users arriving via link may not understand the game concept
- **Recommendation:** Add a one-line tagline below the title (e.g., "Draw it, guess it, laugh at it! The hilarious drawing game")

**1.2 - Room Code Input Lacks Validation Feedback**
- **Issue:** 4-character code input accepts any length, no visual feedback for invalid format
- **Impact:** Users may enter wrong code length and get confused by errors
- **Recommendation:** 
  - Add character counter (e.g., "2/4")
  - Disable/enable JOIN button based on valid input
  - Show inline validation (green checkmark when 4 chars entered)

**1.3 - Loading States Not Visually Clear**
- **Issue:** Button text changes to "CREATING..." or "..." but no spinner or loading indicator
- **Impact:** On slow connections, users may click multiple times
- **Recommendation:** Add spinner icon to button during loading state

#### üìå High Priority Issues (P1)

**1.4 - "How to Play" Could Be More Prominent**
- **Issue:** Info button is relatively small and could be missed
- **Impact:** New players may skip instructions
- **Recommendation:** Make button larger or add a quick "How it works" section directly on landing page

**1.5 - No Share/Invite Link Generation**
- **Issue:** Users can't easily share invite links from landing page
- **Impact:** Players must manually communicate room codes
- **Recommendation:** Add "Copy Link" button that generates shareable URL with room code

### 2. Lobby Screen

#### ‚úÖ Strengths
- Clean player list with ready status indicators
- QR code modal for easy mobile joining
- Host controls (kick, start) are clearly separated
- Good use of disconnection state (opacity reduction)
- Responsive word difficulty settings

#### ‚ö†Ô∏è Critical Issues (P0)

**2.1 - Scrolling Issues in Player List**
- **Issue:** Based on previous conversation history, there were scrolling bugs
- **Status:** May be fixed, but needs verification
- **Impact:** iPhones and other mobile devices may not scroll properly with many players
- **Recommendation:** Test with 12 players on various mobile devices

**2.2 - No Minimum Player Count Warning Until Start Attempted**
- **Issue:** Start button is visible but shows opacity when < 2 players. Error only shows after clicking
- **Impact:** Confusing feedback loop
- **Recommendation:** 
  - Replace START button with status text when < 2 players (e.g., "Waiting for 1 more player...")
  - Only show START when minimum met

**2.3 - Room Code Display Has Poor Mobile Tap Target**
- **Issue:** "CODE: XXXX" clickable area for QR modal is unclear
- **Impact:** Users may not realize it's tappable
- **Recommendation:**
  - Add a QR icon or "tap to share" hint
  - Increase tap target size
  - Add visual affordance (button styling vs plain text)

**2.4 - DrawNGuess Title Appears Twice**
- **Issue:** App title appears in header AND in room info section (lines 246, 254 in Lobby.tsx)
- **Impact:** Redundant, takes up valuable mobile screen space
- **Recommendation:** Remove one instance, prioritize room code visibility

#### üìå High Priority Issues (P1)

**2.5 - No Game Instructions in Lobby**
- **Issue:** No reminder of game rules or flow
- **Impact:** Players ready up without understanding gameplay
- **Recommendation:** Add collapsible "How to Play" section or link back to rules modal

**2.6 - "INVITE" Button Does Nothing?**
- **Issue:** Small "INVITE" button next to room code (line 263) has no onClick handler
- **Impact:** Dead button creates confusion
- **Recommendation:** Either implement share functionality or remove button

**2.7 - Ready Button Hidden for Host**
- **Issue:** Host doesn't need to ready up, but this isn't explained
- **Impact:** Solo host testing might be confused
- **Recommendation:** Show disabled button with tooltip "Host auto-ready"

**2.8 - No Clear Indication of Game Flow**
- **Issue:** Players don't know what happens after START
- **Impact:** Anxiety about unknown next steps
- **Recommendation:** Add text like "Game starts with a 3-second countdown, then you'll draw a secret word!"

### 3. Game - Draw Phase

#### ‚úÖ Strengths
- Canvas has good touch responsiveness
- Color picker is intuitive
- Clear timer display
- "Draw this:" reference box is prominent
- Unsubmit feature allows editing

#### ‚ö†Ô∏è Critical Issues (P0)

**3.1 - Canvas Size on Small Screens**
- **Issue:** 4:3 aspect ratio may be too large on small phones (< 375px width)
- **Impact:** Canvas pushes tools/submit button below fold
- **Recommendation:** 
  - Test on iPhone SE / small Android
  - Consider dynamically reducing canvas height on very small screens
  - Make tools fixed/sticky to prevent scrolling

**3.2 - Accidental Submission Prevention Is Buggy**
- **Issue:** Submit delay calculation shows negative countdown (line 105: `minSubmitDelay - 5000`)
- **Impact:** If `minSubmitDelay` is 5000ms, this shows "Ready in 0s..." instead of "Ready in 5s..."
- **Recommendation:** Fix math to `(minSubmitDelay) / 1000`

**3.3 - No Visual Indication That Drawing Is Saving**
- **Issue:** Drawing data only exports after stroke ends, but no feedback
- **Impact:** Users might worry their art isn't saved
- **Recommendation:** Add subtle "Saved" checkmark or animation after each stroke

**3.4 - Timer Can Hit Zero But Phase Continues**
- **Issue:** Timer counts down but doesn't force submission
- **Impact:** Confusion about what happens at 0s
- **Recommendation:** 
  - Add "TIME'S UP!" overlay when timer expires
  - Auto-submit after timer expires (or show clear "Submit Now or Lose Turn" warning)

#### üìå High Priority Issues (P1)

**3.5 - Brush Width Not Adjustable**
- **Issue:** Line width is hardcoded to 3px (line 90)
- **Impact:** Hard to draw fine details or bold strokes
- **Recommendation:** Add brush size slider (thin/medium/thick)

**3.6 - Limited Color Palette**
- **Issue:** Only 6 colors available
- **Impact:** Limits creative expression
- **Recommendation:** Add 2-3 more common colors (orange, green, brown)

**3.7 - Eraser Size Too Large?**
- **Issue:** Eraser is 20px wide, might be too aggressive
- **Impact:** Hard to erase small mistakes
- **Recommendation:** Make eraser size adjustable or reduce to 15px

**3.8 - No Undo Button**
- **Issue:** Only "Clear" which deletes everything
- **Impact:** One accidental stroke = start over
- **Recommendation:** Add simple undo/redo (track last 5-10 strokes)

### 4. Game - Guess Phase

#### ‚úÖ Strengths
- Large input field with autofocus
- Previous drawing clearly displayed
- Enter key submits guess
- Unsubmit allows correction

#### ‚ö†Ô∏è Critical Issues (P0)

**4.1 - Same Timer Issue as Draw Phase**
- **Issue:** Timer hits 0 but phase continues
- **Impact:** Players wait indefinitely
- **Recommendation:** Auto-submit blank/current guess when time expires

**4.2 - No Character Indicator on Input**
- **Issue:** 50 char limit but no counter shown
- **Impact:** Users hit limit unexpectedly
- **Recommendation:** Add character counter "24/50"

**4.3 - Same Buggy Submit Delay Countdown**
- **Issue:** Math error in line 122 (same as draw phase)
- **Impact:** Shows wrong countdown
- **Recommendation:** Fix formula

#### üìå High Priority Issues (P1)

**4.4 - Keyboard Covers Input on Mobile**
- **Issue:** Virtual keyboard may obscure submit button
- **Impact:** Users can't see submit feedback
- **Recommendation:** 
  - Use fixed positioning for submit area
  - Or allow Enter key to always submit

**4.5 - No Placeholder Variation**
- **Issue:** Static placeholder text
- **Impact:** Missed opportunity for engagement
- **Recommendation:** Rotate fun placeholders ("A cat? A hat? A... ?", "Take your best guess!", etc.)

**4.6 - Drawing Could Be Larger**
- **Issue:** Canvas in reference takes same space as when drawing, but controls not needed
- **Impact:** Hard to see details on small drawings
- **Recommendation:** Make readonly canvas 20% larger

### 5. Book Viewer (Results)

#### ‚úÖ Strengths
- PageView pattern is intuitive
- Shows attribution for each page
- Pagination dots allow jumping to page
- Share functionality generates downloadable image
- Clear ownership ("{Name}'s Book")

#### ‚ö†Ô∏è Critical Issues (P0)

**5.1 - No Swipe Gestures**
- **Issue:** Mobile-first app requires tap navigation instead of swipe
- **Impact:** Feels unnatural on mobile
- **Recommendation:** Implement touch swipe to change pages (left/right)

**5.2 - "Return to Home" Forces Full Reload**
- **Issue:** `window.location.href = '/'` destroys state
- **Impact:** Ugly UX, loses connection gracefully
- **Recommendation:** Use React Router navigation, disconnect socket properly

**5.3 - Share Feature Has No Feedback**
- **Issue:** `generateImageStrip` silently downloads, no loading state
- **Impact:** Users don't know if it worked
- **Recommendation:** 
  - Show "Generating..." loading state
  - Display "Image saved!" toast on completion
  - Handle errors gracefully

**5.4 - Page Counter vs Dots Redundancy**
- **Issue:** Both pagination dots AND "Page X of Y" shown
- **Impact:** Wastes screen space on mobile
- **Recommendation:** Hide counter on mobile, rely on dots

#### üìå High Priority Issues (P1)

**5.5 - Can't View Other Players' Books**
- **Issue:** Only shows your own book
- **Impact:** Missing social payoff - players want to see everyone's results
- **Recommendation:** Add "View All Books" button to see others' chains

**5.6 - No Comparison View**
- **Issue:** Can't see how word evolved alongside secret word
- **Impact:** Hard to catch the "telephone game" effect
- **Recommendation:** Add "Show All Drawings" mode that displays them side-by-side

**5.7 - Share Image Quality Unknown**
- **Issue:** No indication if high-res or compressed
- **Impact:** Blurry shares = bad virality
- **Recommendation:** Document/test output quality, add "HD" badge

## üîÑ General UX Patterns

### P0 Issues

**G.1 - No Offline Detection**
- **Issue:** If connection lost, no clear feedback
- **Impact:** Players think game froze
- **Recommendation:** Add "Connection Lost" overlay with reconnect button

**G.2 - No Reconnection Flow**
- **Issue:** Refresh = kicked from game
- **Impact:** Terrible mobile experience (accidental refreshes common)
- **Recommendation:** Store room/player ID in localStorage, auto-rejoin on reconnect

**G.3 - No Loading States on Phase Transitions**
- **Issue:** Instant phase switches can be jarring
- **Impact:** Confusion about what changed
- **Recommendation:** Add 500ms transition with "Next Round!" animation

### P1 Issues

**G.4 - No Sound Effects**
- **Impact:** Less engaging, especially mobile
- **Recommendation:** Add optional subtle sounds (submit, timer low, phase change)

**G.5 - No Haptic Feedback**
- **Impact:** Missed mobile engagement opportunity
- **Recommendation:** Add vibration on submit, timer warnings (iOS/Android)

**G.6 - No Idle Timeout Warnings**
- **Issue:** Players might AFK without warning
- **Impact:** Game stalls
- **Recommendation:** Show "Are you still there?" before auto-submitting
